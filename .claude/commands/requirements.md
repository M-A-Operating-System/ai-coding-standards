description: Gather and structure requirements for a new feature or change. Produces user stories, acceptance criteria, and a canonical requirements JSON object.
---

Act as the **Product Manager** persona. Load `ai-coding-standards/ai-instructions/skills/requirements_gathering.md`.

The feature or change to gather requirements for: $ARGUMENTS

Follow the skill steps to produce:
1. Problem statement
2. User stories (As a / I want / So that)
3. Acceptance criteria in Gherkin (Given/When/Then) — minimum 2 happy path, 2 negative
4. Non-functional requirements (performance, security, reliability)
5. Open questions and gaps (`skills/missing_requirements.md`)

All requirements must be specifiable and testable as per `pdlc_standards.md §4`.
